《OurMusic 音乐播放器项目开发文档》
一、项目概述
OurMusic 是一款基于 Qt Creator + QML 开发的轻量级音乐播放器，核心目标是实现基础音乐播放、界面交互与基础音乐信息展示功能，打造简洁易用的桌面端音乐播放体验。本项目采用 Qt Quick 框架开发，以 QML 实现界面渲染与交互逻辑，结合 C++ 实现后台数据与业务逻辑，实现界面与逻辑的解耦开发。

二、当前界面功能分析
当前界面为播放器主界面，核心结构与功能模块如下：
顶部导航栏
分为「推荐」「播放器」「我的」三个标签页，当前处于「播放器」页面，采用红色高亮区分当前选中状态，支持标签页切换交互。
音乐信息展示区
专辑封面展示区域（当前为占位图，后续将实现真实封面加载与渲染）
歌曲名称、歌手名称、专辑名称文本信息展示区域，用于展示当前播放音乐的元数据
播放控制区
·播放进度条 / 时间显示（当前显示 0:00 / 3:45，支持进度拖拽控制）
·播放 / 暂停按钮（当前为播放状态图标，点击可切换播放 / 暂停状态）
·上一曲、下一曲切换按钮
·播放列表 / 菜单按钮（三条横线图标，点击可展开播放列表）
·收藏 / 喜欢按钮（心形图标，支持歌曲收藏标记）
三、核心功能模块规划
模块一：基础界面搭建与导航实现
实现目标：完成主界面框架搭建与标签页导航交互，确保界面布局与交互符合设计预期。
具体实现内容：
使用 QML 搭建主窗口，设置窗口标题为 “音乐播放器”，固定初始窗口大小，支持窗口最小化 / 最大化 / 关闭。
实现顶部导航栏（TabBar）：
实现「推荐」「播放器」「我的」三个标签页的切换逻辑，为当前选中标签添加高亮样式（红色背景），未选中标签保持默认样式。
实现标签页切换时的界面内容切换，后续可扩展各标签页独立界面。
完成「播放器」页面的基础布局：专辑封面区域、信息文本区域、播放控制区域的定位与基础样式设置。
模块二：音乐播放核心功能实现
实现目标：实现基础的音频播放控制，支持主流音频格式（MP3、WAV、FLAC 等）。
具体实现内容：
引入 Qt Multimedia 模块，搭建音频播放后台逻辑：实现本地音频文件的加载与播放。
支持播放 / 暂停 / 停止功能，点击播放按钮切换状态并更新图标；实现上一曲 / 下一曲切换，支持循环播放（列表循环 / 单曲循环）/
进度条与时间显示功能：实时更新当前播放进度与总时长；支持拖拽进度条跳转播放位置；处理音频加载过程中的时长获取异常，避免界面显示错误/
音量控制功能：添加音量滑块，支持音量调节与静音切换。
模块三：音乐元数据与界面信息同步
实现目标：实现音频文件元数据解析，将歌曲信息同步展示到界面。
具体实现内容：
实现音频文件元数据解析（使用 Qt Multimedia 的 MediaPlayer 自带元数据接口）：读取歌曲名称、歌手、专辑名称、专辑封面信息；处理无元数据的音频文件，设置默认占位文本与默认封面。
界面信息动态更新：
播放歌曲切换时，自动更新界面上的歌曲 / 歌手 / 专辑名称文本；实现专辑封面的加载与渲染，支持封面自适应缩放与圆角样式；加载失败时显示默认占位图（当前界面的 “专辑封面” 占位图）。
模块四：播放列表与歌曲管理
实现目标：实现本地歌曲导入、播放列表管理与切换功能。
具体实现内容：
本地歌曲导入功能：支持通过文件浏览器选择本地音频文件 / 文件夹，批量导入歌曲到播放列表。
播放列表管理：
实现播放列表的显示、歌曲选择与切换；支持删除播放列表中的歌曲、清空列表功能；实现播放列表与播放控制的联动，点击列表歌曲直接切换播放。
扩展功能：收藏歌曲列表，支持标记喜欢的歌曲并单独管理（对应界面上的心形按钮）。
模块五：界面交互优化与样式完善。
实现目标：优化界面交互体验，完善样式细节，提升用户体验。
具体实现内容：
按钮交互反馈：为所有控制按钮（播放 / 暂停、上一曲 / 下一曲等）添加点击效果、悬停效果。
界面样式优化：
统一界面字体、颜色、间距规范；实现专辑封面的过渡动画（切换歌曲时的淡入淡出效果）；进度条样式优化，匹配整体设计风格。
响应式适配：支持窗口大小调整时，界面元素自动适配布局，避免元素错位或溢出。
模块六：扩展功能
「推荐」页面：实现本地推荐歌曲列表展示，支持点击推荐歌曲直接播放；「我的」页面：展示收藏歌曲列表、本地已导入歌曲列表，支持歌曲管理；主题切换：支持浅色 / 深色主题切换，适配不同使用场景；歌词展示：解析 LRC 歌词文件，实现歌词同步滚动显示。

四、技术实现要点
 （一）基本类列表
User（用户类）：负责用户登录验证、管理个人收藏行为。
FavoriteList（收藏列表类）：管理用户收藏的歌曲，支持添加/移除操作。
MusicSong（歌曲类）：存储歌曲的元数据（歌名、歌手、专辑、封面、时长等）及本地文件路径。
PlayList（播放列表类）：管理待播放的歌曲队列，记录当前播放索引，支持增删歌曲和切换模式。
MusicPlayer（播放器类）：控制音频的播放、暂停、停止、切歌、进度跳转及音量调节。
（二）类之间的关系。
User 拥有一个 FavoriteList（1:1 关联）。
User 可以收藏多首 MusicSong（1:0..* 关联）。
FavoriteList 包含多首 MusicSong（组合关系）。
MusicPlayer 聚合一个 PlayList（播放器包含播放列表，但播放列表可独立存在）。
PlayList 聚合多首 MusicSong（播放列表中可包含 0 到多首歌曲）。
（三）整体业务逻辑
用户启动应用后完成登录（可简化为默认用户），系统加载用户的收藏列表及历史播放列表。
用户通过导入本地歌曲或从收藏中添加歌曲到播放列表。播放器根据播放列表中的当前歌曲执行播放、暂停、切歌等操作，进度和时长实时同步到界面。
用户可通过心形按钮收藏当前播放的歌曲，收藏列表在「我的」页面中展示，并支持快速添加到播放列表或直接播放。
所有后台类通过 Qt 信号与槽机制与 QML 界面通信，实现界面与逻辑的分离。
